# Human Oversight Design

Article 14 — EU AI Act Regulation 2024/1689

## Oversight Architecture

The teacher serves as the human oversight layer for student-AI interaction, consistent with Article 14's requirements:

1. **UNDERSTAND**: Teacher dashboard shows aggregate AI interaction patterns (exploration count, Bloom distribution, topic frequency)
2. **DETECT**: Content reports from students + automated anomaly detection (Perspective API flags)
3. **OVERRIDE**: Teacher can flag/remove specific AI-generated content via report management
4. **HALT**: Teacher can pause AI access for their class by deactivating it
5. **REVERSE**: All AI-generated content is clearly labeled as AI-generated and never presented as authoritative fact

## AI-Generated Disclosure (Article 50 Transparency)

Every AI response includes:
- Visual indicator: "Generated by AI — may contain errors"
- Contextual note: "Discuss with your teacher"
- Metadata: model name, generation timestamp accessible in session data

## Parent Oversight Layer

Parents serve as a secondary oversight layer:
- View child's exploration summaries (anonymized)
- Set daily time limits
- Restrict topic scope
- Pause/resume child's account
- Request complete data deletion

## Escalation Path

```
Student reports content → Teacher reviews → Platform admin (if systemic)
Parent reports concern → Teacher + Platform admin
Automated flag (Perspective API) → Teacher notification within 1 hour (high priority)
```
